Reddit
Estados Unidos, 2005.

¿Quien no conoce Reddit? Es una red social cuya extensa variedad temática la hace especial.
Ésta fue originalemnte una platafroma web sencilla basada en HTML y CSS para su estructura y JavaScript para las funciones internas, pero con el paso del tiempo y el aumento de usuarios, más utilidades se tuvieron que ir agregando, formando una aplicación más extensa. Fue así que llegamos a la actualidad, donde la aplicación utiliza Python, Go, siendo ahora importante para la aplicación, Ruby y, por su puesto, JavaScript para las funciones añadidas a la aplicación.
La aplicación depende de algunas bibliotecas 'básicas' de Python y de JavaScript, así como React de JavaScript, la cual realmente importante para que el usuario pueda utilizarla. Además, ha creado sus propios frameworks como baseplate, experiments y synapse dentro de su repositorio. Incluso, utiliza Kubernetes, un automatizador de escalado y de administración de "aplicaciones en contenedores".
Además, actualmente opera con las base de datos PostgreSQL y Cassandra para el control de la información. Mientras que la primera opera como SQL, la segunda opera como contraparte NoSQL.

Twitch
Estados Unidos, 2007.

La plataforma de Streaming de Videojuegos por defecto.
Originalmente nombrada Justin.tv por los creadores, ésta ya era relativamente compleja, por lo que no tardarían en incorporar funciones útiles. Posteriormente, la plataforma migraría al dominio twitch.tv, donde se dió a conocer por las transmisiones de videojuegos.
Ésta aplicación web, como muchas otras, requiere de estructura en HTML, CSS y JavaScript. Además, utiliza Python en muchas de sus funciones, así como Go. Éstos lenguajes sirven para muchas de las funciones que los usuarios-desarrolladores han añadido y que no pertenecían originalmente a la aplicación, especialmente en el chat.
Para autenticar, Twitch API requeire de OAuth 2.0. Y sus bases de datos están construidas en PostgreSQL y Redis, ésta última NoSQL.
Al igual que muchas otras apliaciones web, ésta utiliza Kubernetes, por su función de escalado.

Librerías Gandhi
México, 2000.

Una de las librerías más grandes de México y Latinoamérica. La librería, originalmente ubicada en el Distrito Federal, hoy Ciudad de México. Ésta fué un espacio cultural y artístico por décadas, sin embargo, a la llegada de los 90, se volvía cada vez más necesario entrar en el mundo digital, y en el 2000, ésta introdujo su sitio web.
Por lo cual, ésta ha contratado distintas empresas para desarrollar la aplicación que le permitiera realizar las ventas de libros en línea, una tarea compleja, pues además requerir la págian web, requería de los servicios de compra, físico y en línea, bases de datos con los libros, envío de libros digitales, y demás utilidades.
Es por eso que, además de HTML y CSS para su estructura, emplea JavaScript en buena cantidad e sus páginas web.
Sin embargo, encontramos que sus funciones están desarrollada por VTEX y Vinneren, empresas de comercio en la nube y tecnología, que se encargan de la mayoría de lo que sucede detrás de la aplicación. Éstas trabajaron a través de productos SaaS con ETL Styrk para la aplicación web de Gandhi. Las bases de datos, particularmente importantes para la libería, fueron creadas a partir de GraphQL. Por último, utiliza React y Node.js.
